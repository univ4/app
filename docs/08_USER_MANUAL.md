## 서비스 소개

univ4는 수험생과 학부모가 함께 사용하는 AI 기반 대입 전략 플랫폼입니다. 어디가(adiga.kr) 199개 대학 입결 데이터, 18개 대학 2027학년도 전형계획, 서울권·수도권 정시 분석 자료를 기반으로 정확한 계산과 근거 있는 조언을 제공합니다.

## 1. 시작하기

### 1.1 회원가입

1. univ4.kr 접속
2. **시작하기** 버튼 클릭
3. 이메일·비밀번호 입력 후 가입
4. 이메일 인증 완료

### 1.2 역할 선택

가입 후 역할을 선택합니다.

| 역할 | 권한 |
|---|---|
| **관리자** | 성적 입력, 규칙 설정, 전략 분석, 모든 기능 |
| **열람자** | 조회, 챗봇 질의, 일정 확인 |

학부모(아빠)가 관리자로 가입 후 수험생(아들), 어머니를 열람자로 초대하는 것을 권장합니다.

### 1.3 학생 프로필 설정

관리자가 최초 1회 설정합니다.

**기본 정보**
- 이름, 학교명, 학년
- 목표 대학 (최대 10개 선택)
- 목표 계열 (인문/자연/예체능)

**수능 선택과목 프로필**
- 국어: 언어와매체 / 화법과작문
- 수학: 미적분 / 기하 / 확률과통계
- 탐구 1·2 과목명
- 제2외국어 (선택)

> 선택과목 프로필은 지원 가능 대학 필터링과 유불리 분석에 자동 적용됩니다.

## 2. 성적 입력

### 2.0 성적 관리 대시보드 (P0-1)

**경로**: `/dashboard/scores` (대시보드 → **성적 관리**)

모의고사·내신을 한 곳에서 입력하고, 탭으로 **모의고사 추이**, **정시 환산**, **교과 내신**, **Z점수** 등을 전환해 볼 수 있습니다. 입력값은 신호등·탐색·시뮬레이터 등 다른 기능에 자동 반영됩니다.

### 2.1 모의고사 성적 입력

**경로**: `/dashboard/scores` → 모의고사 추가

입력 항목:
- 시험 날짜 (예: 2026-06-04)
- 영역별 표준점수 / 백분위 / 등급
  - 국어, 수학, 영어, 탐구1, 탐구2

입력 예시:
```
국어: 표준점수 132, 백분위 96, 1등급
수학: 표준점수 145, 백분위 98, 1등급
영어: 2등급
탐구1 (지구과학I): 표준점수 68, 백분위 94
탐구2 (생명과학I): 표준점수 65, 백분위 90
```

입력 후 자동으로 제공되는 것:
- 영역별 추이 차트 (최근 12회)
- 대학별 환산점수 즉시 계산
- 합격 가능성 신호등 자동 업데이트

### 2.2 내신 성적 입력

**경로**: `/dashboard/scores` → 내신 추가

입력 항목 (과목별, 나이스 열 순서에 맞춤):
- 학년·학기 (`1-1` … `3-2`), 과목 구분(보통교과 / 진로선택과목 / 체육·예술)
- 과목명, 단위수(이수학점)
- 합계(지필+수행 가중합산), 원점수, 과목평균, 표준편차, 수강자수
- 보통교과만: 석차, 전체인원, 석차등급
- 성취도(보통교과는 선택, 진로선택·체육·예술은 필수)

> 과목 구분에 따라 입력란이 자동으로 바뀝니다. 석차등급·성취도 반영 방식은 대학 전형 규칙을 따릅니다.

입력 후 자동으로 제공되는 것:
- **Z점수·고교 수준 참고 (P1-2)** — 같은 화면(`/dashboard/scores`) **내신** 탭 하단 **Z점수** 섹션에서 과목별 Z·평균 Z 요약 확인 (`GET /api/scores/zscore`)
- 대학별 실질 내신 등급 자동 산출
- 학생부교과 합격 가능성 신호등 자동 업데이트

### 2.3 나이스(NEIS)에서 데이터 가져오기

나이스(https://www.neis.go.kr)에서 아래 자료를 다운로드하여
app/에 직접 입력하거나 파일로 업로드할 수 있습니다.

#### 내신 성적 입력

1. 나이스 → 학교생활기록부 → 교과학습발달상황 출력
2. 과목별 원점수/평균/표준편차/단위수/석차등급 확인
3. app/ → 성적 관리 → 내신 추가에서 과목별 입력
   (고1 1학기부터 현재까지 전체 입력 권장)

#### 성적표 자동 파싱 (P2-11)

**경로**: `/dashboard/scores` → **이미지로 입력** 탭

나이스(NEIS) 학생 성적 확인서 **PNG/JPEG** 스크린샷을 올리면 Claude Vision으로 과목을 추출합니다. 미리보기 표에서 값을 고친 뒤 **확인 후 저장**을 누르면 `academic_records`에 내신 행이 upsert됩니다. API는 [`POST /api/scores/parse-image`](./04_API_SPEC.md)(multipart·선택 `dry_run`, 또는 수정본 JSON 저장)를 사용합니다.

직접 입력이 필요하면 **내신** 탭에서 과목별로 입력할 수 있습니다.
추후 업데이트에서 PDF를 업로드하면 자동으로 파싱·입력되는
기능이 추가될 예정입니다.

지금은 아래 순서로 수동 입력하세요:

1. 나이스 성적표 PDF 열기
2. 성적 관리 → 내신 추가
3. 과목별 원점수/평균/표준편차/단위수/석차등급 입력
4. 학기별로 반복 (1-1 → 1-2 → 2-1 순서 권장)

#### 모의고사 성적 입력

1. 학교 배부 모의고사 성적표 확인
   (3월·6월·9월 전국연합, 수능 모의평가 등)
2. app/ → 성적 관리 → 모의고사 추가에서 회차별 입력
3. 고1~3 전체 입력 시 영역별 추이 차트 자동 생성

#### 생활기록부 업로드 (세특 분석용)

1. 나이스 → 학교생활기록부 PDF 출력
2. app/ → 생기부 관리 → 파일 업로드
3. AI가 항목별 내용을 자동 분석
   - 세특 글자수·공백 탐지
   - 학업역량·진로역량·공동체역량 분석
   - 목표 대학 인재상 대비 Gap 분석

> 나이스 데이터는 본인 계정에만 저장되며
> 외부로 전송되지 않습니다.
> 생활기록부는 개인 민감정보이므로
> 본인 또는 보호자만 업로드하세요.

> **현재(2026-03-30):** NEIS에 맞춘 **구조화 입력**(세특·창체·수상·행동특성·출결·봉사·독서·자격증·학교폭력 등 탭)으로 DB에 저장할 수 있다. 구조화 데이터를 임베딩한 **`student_record_chunks`** 적재 후, 대시보드 **학종 역량 분석** (`/dashboard/hakjong-analysis`)에서 학업·진로·공동체 역량 분석을 스트리밍으로 받을 수 있다([`docs/04_API_SPEC.md`](./04_API_SPEC.md) `POST /api/student-record/analyze`). **세특 Gap 분석** (`/dashboard/gap-analysis`)에서는 목표 대학 전형계획 청크와 세특을 함께 참고해 강점·보완·주차별 액션 플랜을 스트리밍으로 받을 수 있다([`docs/04_API_SPEC.md`](./04_API_SPEC.md) `POST /api/student-record/gap-analysis`; 문장 대필 없음). **탐구 주제 추천** (`/dashboard/research-topics`)에서는 세특 청크(중복 방지)·전형계획 청크를 근거로 탐구 주제·난이도·소요 기간을 스트리밍 제안한다([`docs/04_API_SPEC.md`](./04_API_SPEC.md) `POST /api/research-topics`; 문장 대필 없음).

## 3. 합격 가능성 신호등 (P0-4 · P1-17)

univ4의 핵심 기능입니다. 입력된 성적과 199개 대학 입결 데이터를 비교하여 합격 가능성을 자동 분류합니다.

### 3.1 신호등 기준

| 등급 | 기준 | 의미 |
|---|---|---|
| 🟢 안정 | 내 점수 > 컷오프 + 5점 | 합격 가능성 높음 |
| 🟡 적정 | 컷오프 ± 5점 범위 | 경쟁 필요 |
| 🔴 도전 | 내 점수 < 컷오프 - 5점 | 상향 지원 |

### 3.2 전체 대학 자동 스캔

**경로**: 대시보드 → **신호등** (`/dashboard/signals`)

표가 로드되면 입결·성적 기준으로 행별 신호등·대표 합격 확률(% 구간)이 표시됩니다. **의대 증원 보정** 등은 쿼리/토글로 반영됩니다(앱 UI 기준). API로 동일하게 조회할 때는 `GET /api/signals?medShift=1` 로 행별 `med_shift_coeff` 컷 반영 여부를 켤 수 있으며, 응답 `meta.med_shift_enabled`로 확인합니다([`docs/04_API_SPEC.md`](./04_API_SPEC.md) §2).

**전체 대학 스캔** 버튼을 클릭하면 199개 대학 전체에 대해 3초 이내로 신호등이 표시됩니다.

필터 옵션:
- 전형 유형: 학생부교과 / 학생부종합 / 정시
- 신호등: 안정만 / 적정만 / 도전만
- 지역: 서울 / 수도권 / 전국
- 수능최저: 있음 / 없음
- 면접: 있음 / 없음

결과 표시:
```
대학명    전형명           계열   컷오프  내 점수  신호등
서강대    지역균형          자연   1.8     2.1     🟡 적정
성균관대   학생부교과(추천)  자연   2.0     2.1     🟡 적정
한양대    지역균형          자연   2.3     2.1     🟢 안정
```

### 3.3 보정 계수 적용

2026학년도 의대 증원으로 인한 이월 효과를 반영할 수 있습니다.

**설정**: 신호등 → 보정 설정 → 의대 증원 보정 ON/OFF

보정 ON 시 커트라인이 하향 조정되어 신호등이 유리하게 변할 수 있습니다. 보정 전/후 결과를 나란히 비교할 수 있습니다.

### 3.4 논술전형 실질 경쟁률 판독기 (P1-3)

**경로**: 대시보드 → **논술 경쟁률 판독기** (`/dashboard/nulsul`)

`admission_records`에 있는 **논술전형** 행을 고르면 명목 경쟁률이 자동으로 채워집니다(없으면 직접 입력). 수능최저 충족률(0~100%)과 결시율(기본 10%)을 넣으면 **실질 경쟁률**, **명목 경쟁률**, **차이(명목−실질)**가 함께 표시됩니다. 여러 대학을 **비교 표에 추가**하면 한 테이블에서 나란히 볼 수 있습니다. 수식·API는 [`docs/04_API_SPEC.md`](./04_API_SPEC.md)의 `GET /api/nulsul`·Track 1 `calcRealCompetitionRate` 설명을 따릅니다.

### 3.5 조기 설계 로드맵 (P3-3)

**경로**: 대시보드 → **조기 설계 로드맵** (`/dashboard/early-roadmap`)

고1·고2 학생 기준으로 현재 학년·학기·목표 대학 수준·계열을 선택한 뒤 **로드맵 적용**을 누르면, 네 학기 단위로 내신 목표·우선순위·권장 활동·주요 마일스톤이 표시됩니다. 현재 학기에 해당하는 카드가 강조됩니다. 합격을 보장하지 않으며, 참고용입니다. API는 [`docs/04_API_SPEC.md`](./04_API_SPEC.md) `POST /api/early-roadmap`·Track 1 `calcEarlyRoadmap`을 따릅니다.

## 4. 정시 환산점수 계산기 (P0-2)

### 4.1 대학별 환산점수 자동 계산

**경로**: `/dashboard/scores` → **정시 계산** 탭

모의고사 성적이 입력된 상태에서 위 탭을 열면 18개 대학의 환산점수가 자동으로 계산됩니다.

적용 규칙 (대학별 자동 로드):
- 수능 영역별 반영비율 (국/수/탐/영)
- 영어 등급 환산표
- 과탐II 가산점 (성균관대 5%, 연세대 3% 등)
- 탐구 변환표준점수

계산 결과 예시:
```
서강대 자연계열
  국어 (20%): 132 × 0.20 = 26.4
  수학 (35%): 145 × 0.35 = 50.8
  탐구 (45%): 변환표준점수 적용 = 58.7
  영어 2등급 환산: 96점
  총점: 1,000점 만점 기준 932.4점
  
전년도 컷오프: 928.6점
신호등: 🟡 적정 (+3.8점)
```

### 4.2 과탐 가산점 시뮬레이터

과탐II 선택 여부에 따른 환산점수 변화를 즉시 확인합니다.

**경로**: 대시보드 → **과탐 조합 시뮬레이터** (`/dashboard/science-combo`) — 탐구1·2 조합별 과탐Ⅱ 가산 적용 대학 비교 (P3-4)

```
[과탐II 가산점 적용 OFF]  서강대: 932.4점 🟡 적정
[과탐II 가산점 적용 ON ]  서강대: 941.2점 🟢 안정  (+8.8점)

가산점 적용 대학:

## 13. 입시 D-Day 캘린더 (P0-5 · P1-12)

**경로**: 대시보드 → **캘린더** (`/dashboard/calendar`)

2027학년도 주요 입시 일정과 D-Day를 한눈에 확인합니다. 최초 접속 시 기본 일정 4건(수시·수능·정시 등)이 자동 생성될 수 있으며, Admin은 일정 추가·수정·삭제가 가능하고 Viewer는 조회만 합니다(`docs/04_API_SPEC.md` §5b).

```
오늘 기준 주요 일정:

D-162  수시 원서접수 시작    2026-09-07

  1. 수능최저 기준 과목별 목표 등급 재확인

화면에서는 **역산 TO-DO** 섹션과 대시보드 상단 카드에, 등록된 일정 유형(`원서접수`·`수능`·`정시`)과 오늘 기준 D-Day에 맞춰 위와 같은 체크리스트가 자동 생성됩니다. 완료 체크는 브라우저(기기)에만 저장됩니다.

역할별 알림:
- 관리자: D-30, D-7, D-1 중요 일정 알림
- 열람자: D-7, D-1 알림

## 15. 자소서 코치 (P1-6)

**경로**: `/dashboard/personal-statement`

지원 대학·전형·문항에 맞춰 자기소개서 초안을 작성하고, 생기부·역량 근거를 바탕으로 한 피드백과 글자수 점검을 받을 수 있습니다. 문장 대필은 제공하지 않으며, API는 [`docs/04_API_SPEC.md`](./04_API_SPEC.md)의 `POST /api/personal-statement/feedback` 등을 따릅니다.

사용 방법:
1. 대시보드에서 **자소서 코치**로 이동한다.
2. 전형·문항·본문 초안을 입력한다.
3. 피드백 요청 후 역량별 코멘트와 보완 포인트를 확인한다.

주의사항 (있으면):
- 합격·서류 평가 결과를 보장하지 않으며, 최종 문장은 본인이 검토·수정한다.

## 16. AI 모의 면접 코치 (P1-9)

**경로**: `/dashboard/mock-interview`

생활기록부와 전형계획(요강) 청크를 참고해 면접 질문을 생성하고, 답변에 대한 피드백을 스트리밍으로 받을 수 있습니다. API는 [`docs/04_API_SPEC.md`](./04_API_SPEC.md)의 `/api/mock-interview/*` 설명을 따릅니다.

사용 방법:
1. 대시보드에서 **모의 면접**으로 이동한다.
2. 면접 맥락(전형·목표 대학 등)과 답변을 입력한다.
3. 생성된 질문·피드백을 바탕으로 답변을 다듬는다.

주의사항 (있으면):
- 실제 면접과 동일하지 않으며, 학교 공식 안내·모의면접 기회와 병행하는 것이 좋다.

## 17. 자주 묻는 질문

**Q. 데이터는 얼마나 최신인가요?**
입결 데이터는 2026학년도(2025 수능) 기준이며, 전형계획은 2027학년도 기준입니다. 매년 4~5월 새 데이터로 업데이트됩니다.

**Q. 합격을 보장하나요?**
univ4는 데이터 기반 참고 정보를 제공합니다. 실제 합격 여부는 당해 연도 경쟁 상황, 수능 결과, 서류 평가 등 다양한 요인에 따라 달라집니다.

**Q. 챗봇이 틀린 답변을 하면 어떻게 하나요?**
챗봇은 반드시 출처를 표기합니다. 출처가 없거나 "확인 불가"로 답변한 경우 해당 대학 입학처에 직접 문의하세요.

**Q. 내 성적 데이터는 안전한가요?**
모든 데이터는 본인 계정에만 접근 가능하며 타인과 공유되지 않습니다. Supabase RLS 정책으로 계정별 데이터가 완전히 격리됩니다.
