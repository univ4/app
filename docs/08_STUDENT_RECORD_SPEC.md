# 생활기록부 입력 폼 설계 (Student Record Form Spec)

**문서 목적:** NEIS 생활기록부에 대응하는 **입력 폼·데이터 구조·우선순위·RAG 적재**를 구현 전에 고정한다.  
**구현 상태:** 본 문서는 **설계 전용**이며, UI·API·마이그레이션은 별도 작업에서 수행한다.

**관련 문서**

- 교과 성적: `academic_records` — [`docs/03_DATA_MODEL.md`](./03_DATA_MODEL.md), [`docs/03_DB_SCHEMA.md`](./03_DB_SCHEMA.md)
- 로드맵: [`docs/05_ROADMAP.md`](./05_ROADMAP.md)

---

## 1. 인적·학적사항

### 1.1 구조화 입력

| 필드 | 입력 방식 | 비고 |
|------|-----------|------|
| 성명 | 단일 행 텍스트 | NEIS 표기와 동일 권장 |
| 성별 | 선택 (예: 남/여/기재안함) | 코드값은 구현 시 enum으로 고정 |
| 학교명 | 단일 행 텍스트 또는 선택(시드 연동 시) | |
| 학년 | 선택 (1 / 2 / 3) | 입력 시점 기준 학년 |

### 1.2 자유 서술

| 필드 | 입력 방식 |
|------|-----------|
| 학적사항 | 다줄 텍스트 — 입학일, 전학 이력 등 |
| 특기사항 | 다줄 텍스트 |

### 1.3 데이터 보관 (설계 메모)

구조화 필드는 **`students` 테이블 확장** 또는 **`student_profiles` 등 전용 1:1 테이블**로 보관하는 방안 중 하나로 DDL 확정 시 [`docs/03_DATA_MODEL.md`](./03_DATA_MODEL.md)에 반영한다. 본 스펙은 폼 항목만 정의한다.

## 2. 출결상황

**단위:** 학년별 1~3학년, 학년당 1행(또는 학년 탭).

### 2.1 구조화 필드 (학년별)

| 필드 그룹 | 하위 항목 |
|-----------|-----------|
| 수업일수 | 정수 |
| 결석 | 질병 / 미인정 / 기타 (각 정수) |
| 지각 | 질병 / 미인정 / 기타 |
| 조퇴 | 질병 / 미인정 / 기타 |
| 결과 | 질병 / 미인정 / 기타 |

### 2.2 자유 서술

| 필드 | 입력 방식 |
|------|-----------|
| 특기사항 | 다줄 텍스트 |

### 2.3 검증·UX 메모

- 일수·횟수는 0 이상 정수, 상한은 NEIS 제약에 맞춤.
- “결과”는 NEIS 용어에 맞춰 라벨 확정(구현 시).

### 4.1 자격증·인증 (반복)

| 필드 | 입력 방식 |

| 취득연월일 | 날짜 |

### 4.2 국가직무능력표준(NCS) 이수상황 (반복)

| 필드 | 입력 방식 |
|------|-----------|
| 학년 | 선택 (1 / 2 / 3) |
| 학기 | 선택 (1 / 2) |

## DB 테이블 설계 (신규)

아래 테이블은 **생활기록부 스펙 구현 시** 추가를 전제로 한다. 컬럼 상세·FK·제약은 마이그레이션 작성 시 [`docs/03_DATA_MODEL.md`](./03_DATA_MODEL.md)에 반영한다.

| 테이블명 | 용도 |
|----------|------|
| `student_awards` | 수상경력 (§3) |
| `student_attendance` | 출결상황 (§2) |
| `student_certificates` | 자격증 및 인증 (§4.1) |
| `student_school_violence` | 학교폭력 조치사항 (§5) |
| `student_activities` | 창의적 체험활동 — 자율/동아리/진로 (§6.1) |
| `student_volunteer` | 봉사활동 (§6.2) |
| `student_subject_notes` | 세부능력 및 특기사항 (§7.2) |
| `student_reading` | 독서활동 (§8) |
| `student_behavior` | 행동특성 및 종합의견 (§9) |

### 공통 컬럼 (모든 신규 테이블)

| 컬럼 | 설명 |
|------|------|
| `id` | PK (uuid 권장) |
| `student_id` | FK → `students(id)` |
| `created_at` | 생성 시각 |
| `updated_at` | 수정 시각 |

### RLS (정책 방향)

- **authenticated:** 자신의 `student_id`에 대해 **읽기** 허용(가족·Viewer 스코프는 기존 `students` 정책과 정합).
- **쓰기:** **Admin(아버지 데이터 입력 역할)** 중심 — `service_role` 또는 `admin` 클레임 등 기존 앱 역할 모델에 맞춰 INSERT/UPDATE/DELETE 허용.  
  정확한 표현식은 마이그레이션과 동일 작업에서 [`docs/03_DB_SCHEMA.md`](./03_DB_SCHEMA.md)에 기술한다.

### NCS 이수 (§4.2)

- 별도 테이블 `student_ncqs`(가칭) 또는 `student_certificates` 통합 — DDL 확정 시 위 문서들에 추가한다.

## 구현 우선순위

### P1 — 학종 분석·RAG에 필수

1. `student_subject_notes` — 세부능력 및 특기사항  
2. `student_activities` — 창의적 체험활동  
3. `student_awards` — 수상경력  
4. `student_behavior` — 행동특성 및 종합의견  

### P2 — 보완

5. `student_attendance` — 출결  
6. `student_volunteer` — 봉사활동  
7. `student_reading` — 독서활동  

### P3 — 참고·민감

8. `student_certificates` — 자격증·인증 (+ NCS는 DDL에서 분리 여부 확정)  
9. `student_school_violence` — 학교폭력 조치사항

## RAG 적재 대상

다음 텍스트는 **pgvector 임베딩 후 RAG**에 사용한다. (`guideline_chunks`와 **별도**)

| 소스 | 테이블 | 비고 |
|------|--------|------|
| 세특 | `student_subject_notes` | 과목별 서술 |
| 창체 특기사항 | `student_activities` | 영역·학년별 `특기사항` 필드 |
| 행동특성 | `student_behavior` | 학년별 종합의견 |

### 적재 테이블

- **`student_record_chunks`** — 모집요강 RAG용 `guideline_chunks`와 분리하여 학생별 근거 검색에 사용. DDL·RLS·`match_student_record_chunks` RPC는 `supabase/migrations/20260330250000_student_record_chunks.sql`, SQL Editor용 함수만은 `scripts/ingest/match_student_record_chunks.sql`, 임베딩 적재는 `scripts/ingest/embed_student_record.ts` ([`scripts/ingest/README.md`](../scripts/ingest/README.md)).

### 청크 메타데이터 (예시)

| 키 | 설명 |
|----|------|
| `student_id` | 학생 식별 |
| `grade` | 학년 |
| `subject_name` | 세특 과목명 등 |
| `activity_type` | 창체 영역(자율/동아리/진로) |
| `section` | `세특` / `창체` / `행동특성` (적재 스크립트·`metadata` 키와 동일) |

임베딩 모델·청크 분할 규칙은 [`docs/05_AI_PIPELINE.md`](./05_AI_PIPELINE.md)와 정합시킨다.

## API·폼 구현 시 체크리스트 (참고)

- 장문 필드(3000자) — 프론트·서버 양쪽 길이 검증.
